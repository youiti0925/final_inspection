## 1. アプリの起動モードについて

このアプリには **2つの起動モード** があります。

| モード | バックエンド | 特徴 |
|--------|------------|------|
| **Firebase版（現在の設定）** | Google Firebase | インターネット必須。DB設定不要。 |
| **PocketBase版** | ローカルサーバー | オフライン動作可能。初回設定あり。 |

どちらのモードかは `src/main.jsx` の内容で決まります：

```js
// Firebase版（この行が有効ならFirebase）
import App from './App.firebase.jsx'

// PocketBase版（この行が有効ならPocketBase）
// import App from './App.jsx'
```

**現在は Firebase版が有効** になっています。

## 2. start.bat が自動でやること

`start.bat` をダブルクリックすると、以下が **自動的に** 実行されます。

```
[STEP 1] Node.js の確認
    └─ npmコマンドが使えるか確認
    └─ ない場合 → ブラウザでhttps://nodejs.org/を開いてエラー停止

[STEP 2] パッケージの確認
    └─ node_modules フォルダがあるか確認
    └─ ない場合 → npm install を自動実行（初回のみ数分かかる）

[STEP 3] 起動モード判定
    └─ src/main.jsx を読んでFirebase版かPocketBase版か自動判定

▼ Firebase版の場合
    └─ ブラウザ自動起動 → 完了

▼ PocketBase版の場合（続き）

[STEP 4] PocketBase の確認
    └─ pocketbase.exe があるか確認
    └─ ない場合 → GitHubから自動ダウンロード＆解凍

[PocketBase起動]
    └─ 黒いウィンドウが開いてサーバーが起動

[初回のみ] セットアップウィザード
    └─ pb_data フォルダがない = 初回と判断
    └─ ブラウザで管理者登録画面を自動表示
    └─ Email・パスワードを入力させる
    └─ DBコレクションを自動初期化

[フロントエンド起動]
    └─ ブラウザ自動起動 → 完了
```

## 3. start.bat の全コード解説

以下が `start.bat` の全コードです。各行に何をしているか説明します。

```batch
@echo off
```
> コマンド自体を画面に表示しない設定。これがないと入力したコマンドが全部画面に出て見づらい。

```batch
chcp 65001 >nul 2>&1
```
> コンソールの文字コードをUTF-8に切り替える。これがないと日本語が文字化けする。

```batch
cd /d "%~dp0"
```
> このbatファイルがある場所にカレントディレクトリを移動する。どこからダブルクリックしても正しいフォルダで動く。

```batch
setlocal enabledelayedexpansion
```
> `!変数名!` という書き方を有効にする。`if` ブロックの中で変数の値が正しく読めるようにするために必要。

```batch
title 検査アプリ起動
```
> コンソールウィンドウのタイトルバーに「検査アプリ起動」と表示する。

---

### STEP 1: Node.js の確認

```batch
echo [STEP 1] Node.js の確認...
where npm >nul 2>&1
if !errorlevel! neq 0 (
    echo  *** エラー: Node.js がインストールされていません ***
    echo  インストール手順:
    echo    1. https://nodejs.org/ にアクセス
    echo    2. "LTS" ボタンをクリックしてダウンロード
    echo    3. インストーラーを実行する
    echo    4. インストール完了後、このファイルを再実行してください
    start "" "https://nodejs.org/"
    goto :ERROR_EXIT
)
for /f "tokens=*" %%v in ('npm --version 2^>nul') do set NPM_VER=%%v
echo         OK  (npm v!NPM_VER!)
```

- `where npm` → npmコマンドが存在するか検索。見つからなければエラーコード1が返る
- `!errorlevel! neq 0` → エラーコードが0以外 = npmがない
- npmがない場合：エラーメッセージを表示してブラウザでNode.jsのサイトを開く
- npmがある場合：バージョンを取得して `OK (npm v11.x.x)` と表示

---

### STEP 2: パッケージの確認

```batch
echo [STEP 2] パッケージの確認...
if not exist "node_modules\" (
    echo         node_modules が見つかりません。npm install を実行します。
    echo         初回は数分かかります。このまま待ってください
    echo.
    npm install
    set NPM_ERR=!errorlevel!
    echo.
    if !NPM_ERR! neq 0 (
        echo  *** エラー: npm install が失敗しました exitcode=!NPM_ERR! ***
        echo  よくある原因:
        echo    - インターネットに接続されていない
        echo    - ファイアウォール/プロキシがブロックしている
        echo    - package.json が壊れている
        goto :ERROR_EXIT
    )
    echo         OK  インストール完了
) else (
    echo         OK
)
```

- `node_modules\` フォルダがなければ初回と判断して `npm install` を自動実行
- `set NPM_ERR=!errorlevel!` → npm installの結果コードを保存
- インストール失敗の場合はエラー表示して停止
- 2回目以降はフォルダがあるので `OK` とだけ表示してスキップ

---

### STEP 3: 起動モード判定

```batch
echo [STEP 3] 起動モードの確認...
set USE_FIREBASE=0

findstr /R "^import App from.*firebase" src\main.jsx >nul 2>&1
if not errorlevel 1 set USE_FIREBASE=1

if !USE_FIREBASE!==1 (
    echo         Firebase版 - PocketBase は使用しません
    goto :START_VITE
)
echo         PocketBase版
```

- `findstr /R "^import App from.*firebase"` → `src\main.jsx` の中に `import App from '...firebase...'` という行があるか正規表現で検索
- 見つかれば Firebase版と判断 → STEP 4を飛ばして直接Vite起動へ
- 見つからなければ PocketBase版と判断 → STEP 4に進む

---

### STEP 4: PocketBase の確認・自動ダウンロード（PocketBase版のみ）

```batch
echo [STEP 4] PocketBase の確認...

if not exist "pocketbase.exe" (
    echo         pocketbase.exe が見つかりません。自動ダウンロードを開始します。
    powershell -NoProfile -ExecutionPolicy Bypass -Command "..."

    if not exist "pocketbase.exe" (
        echo  *** エラー: PocketBase のダウンロードに失敗しました ***
        echo  手動でダウンロードしてください:
        echo    https://pocketbase.io/docs/
        start "" "https://pocketbase.io/docs/"
        goto :ERROR_EXIT
    )
)
for /f "tokens=*" %%v in ('.\pocketbase.exe --version 2^>nul') do set PB_VER=%%v
echo         OK  (!PB_VER!)
```

- `pocketbase.exe` がなければ PowerShell を使って GitHub から最新版を自動ダウンロード
- PowerShell の中身：
  1. GitHub API (`api.github.com/repos/pocketbase/pocketbase/releases/latest`) で最新バージョンを取得
  2. `windows_amd64` のzipを検索してダウンロード
  3. その場に解凍して zip を削除
- ダウンロード後も `pocketbase.exe` がなければエラー停止

---

### PocketBase 起動

```batch
set IS_FIRST_RUN=0
if not exist "pb_data\" set IS_FIRST_RUN=1

echo  PocketBase を起動しています...
start "PocketBase Server" cmd /k "title [PocketBase] && .\pocketbase.exe serve --http=0.0.0.0:8090"

if !IS_FIRST_RUN!==0 goto :START_VITE
```

- `pb_data\` フォルダがなければ初回と判断 (`IS_FIRST_RUN=1`)
- 別の黒いウィンドウで PocketBase サーバーを起動（ポート8090）
- `--http=0.0.0.0:8090` → ネットワーク上の他のPCからもアクセス可能
- 初回でなければ（`IS_FIRST_RUN=0`）セットアップをスキップしてVite起動へ

---

### 初回セットアップウィザード（PocketBase版・初回のみ）

```batch
echo  PocketBase の起動を待っています... (4秒)
timeout /t 4 /nobreak >nul

echo  ブラウザで管理者アカウント作成画面を開きます。
start "" "http://localhost:8090/_/"
echo   手順:
echo     1. ブラウザが開いたら Email を入力
echo     2. Password を設定 (8文字以上)
echo     3. "Create and login" ボタンを押す
echo  アカウントの作成が完了したら Enter キーを押してください。
pause >nul
```

- PocketBase が完全に起動するまで4秒待つ
- ブラウザで `http://localhost:8090/_/` を開く（PocketBase管理画面）
- ユーザーが管理者アカウントを作成したら Enter を押して次へ

```batch
:SETUP_INPUT
set /p PB_EMAIL=  管理者メールアドレス:
if "!PB_EMAIL!"=="" (
    echo  入力されていません。もう一度入力してください。
    goto :SETUP_INPUT
)

（パスワード入力：PowerShellで非表示入力）

node setup_db_v3.mjs "!PB_EMAIL!" "!PB_PASSWORD!"
set SETUP_ERR=!errorlevel!
```

- コンソールでメールアドレスとパスワードを入力させる
- パスワードは PowerShell の `Read-Host -AsSecureString` を使って入力中は `*` で非表示
- `node setup_db_v3.mjs` でDBコレクション（lots, workers, settings, target_time_history）を自動作成

---

### Vite（フロントエンド）起動

```batch
:START_VITE
echo  フロントエンドを起動しています...
start "Vite Dev Server" cmd /k "title [Vite] && npm run dev -- --host --open"

echo =============================================
echo   起動が完了しました！
echo   [PocketBase] と [Vite] の黒いウィンドウは
echo   アプリ使用中は閉じないでください。
echo =============================================
```

- 別の黒いウィンドウで `npm run dev` を実行
- `--host` → ネットワーク上の他のPCからもアクセス可能
- `--open` → ブラウザが自動で開く
- **この黒いウィンドウを閉じるとアプリが止まります**

---

### エラー終了・完了

```batch
:ERROR_EXIT
echo  問題が解決したら、このファイルを再実行してください。

:DONE
pause
```

- 正常終了・エラー終了どちらも最後に `pause` でウィンドウが止まる
- キーを押すとウィンドウが閉じる

## 4. 手動セットアップ手順（自動が失敗した場合）

### 4-1. Node.js のインストール（手動）

1. ブラウザで `https://nodejs.org/` を開く
2. **LTS** と書いてある大きいボタンをクリックしてダウンロード
3. ダウンロードした `.msi` ファイルを実行してインストール
4. インストール中の選択肢はすべてデフォルトのままで OK
5. インストール後、**PCを再起動するか新しいコマンドプロンプトを開く**
6. `start.bat` を再度ダブルクリック

---

### 4-2. npm install の手動実行

`start.bat` と同じフォルダで **右クリック → ターミナルで開く** または **コマンドプロンプトを開いて移動** し：

```
npm install
```

エラーが出た場合：
```
npm install --legacy-peer-deps
```

---

### 4-3. PocketBase の手動ダウンロード（PocketBase版のみ）

1. ブラウザで `https://github.com/pocketbase/pocketbase/releases/latest` を開く
2. `Assets` の中から `pocketbase_X.X.X_windows_amd64.zip` をダウンロード
3. zipを解凍して中の `pocketbase.exe` を **`start.bat` と同じフォルダ** に置く
4. `start.bat` を再度ダブルクリック

---

### 4-4. PocketBase 管理者アカウントの手動作成（PocketBase版・初回のみ）

1. まず `pocketbase.exe` を直接ダブルクリックして起動（または下記コマンド）
   ```
   .\pocketbase.exe serve
   ```
2. ブラウザで `http://localhost:8090/_/` を開く
3. Email（例：`admin@example.com`）と Password（8文字以上）を入力
4. **Create and login** ボタンをクリック

---

### 4-5. DBコレクションの手動初期化（PocketBase版・初回のみ）

管理者アカウント作成後、`start.bat` と同じフォルダでコマンドプロンプトを開き：

```
node setup_db_v3.mjs admin@example.com yourpassword
```

（メールアドレスとパスワードは4-4で設定したもの）

成功すると：
```
✓ lots コレクション作成
✓ workers コレクション作成
✓ settings コレクション作成
✓ target_time_history コレクション作成
✓ 接続テスト完了
```
のように表示される。

---

### 4-6. Vite（フロントエンド）の手動起動

`start.bat` と同じフォルダのコマンドプロンプトで：

```
npm run dev -- --host --open
```

ブラウザが自動で開いてアプリが表示される。
**このコマンドプロンプトは閉じないこと（閉じるとアプリが止まる）**

## 5. よくあるエラーと対処法

### 「npm は内部コマンドまたは外部コマンドとして認識されていません」
→ **Node.js がインストールされていない。** 4-1を参照。

### 「ERR_MODULE_NOT_FOUND」や npm install エラー
→ **インターネット接続を確認。** または：
```
npm install --legacy-peer-deps
```

### ターミナルが一瞬開いてすぐ閉じる
→ `start.bat` の書き方に問題がある可能性。PowerShellで確認：
```powershell
cmd /c "C:\(フォルダパス)\start.bat"
```
エラーメッセージが残るので原因を特定できる。

### PocketBase 画面が開かない（初回セットアップ）
→ 4秒では起動が間に合わないことがある。手動で `http://localhost:8090/_/` をブラウザで開く。

### DB初期化が失敗する
→ パスワードが8文字未満か、PocketBaseがまだ起動中の可能性。少し待ってから再試行。または4-5を手動実行。

### Firebase版なのにエラーが出る
→ `src/main.jsx` を確認。以下の行が `//` なしで有効になっているか確認：
```js
import App from './App.firebase.jsx'
```

### ポート競合エラー（PocketBase）
→ 別のPocketBaseが起動中。タスクマネージャーで `pocketbase.exe` を終了してから再起動。

## まとめ：通常の起動手順

**Firebase版（現在の設定）の場合：**
1. `start.bat` をダブルクリック
2. 初回のみ `npm install` が自動実行される（数分待つ）
3. ブラウザが開いてアプリ起動 ✓

**PocketBase版の場合（初回）：**
1. `start.bat` をダブルクリック
2. 初回のみ `npm install` が自動実行される
3. PocketBase が自動ダウンロードされる（初回のみ）
4. ブラウザで管理者アカウントを作成する（画面の指示通り）
5. Enter を押してメール・パスワードを入力
6. DBが自動初期化される
7. ブラウザが開いてアプリ起動 ✓

**PocketBase版の場合（2回目以降）：**
1. `start.bat` をダブルクリック
2. 自動でPocketBase起動 + ブラウザ開く ✓
