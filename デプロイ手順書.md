# アプリ公開（デプロイ）手順書

## そもそも何をしているのか？

今まではパソコンの中だけでアプリを動かしていました。
それをインターネット上のサーバー（Firebase Hosting）に置くことで、
タブレットやスマホからURLでアクセスできるようにしています。

```
【今まで】
パソコン → npm run dev → パソコンのブラウザでしか見れない

【デプロイ後】
パソコン → ビルド → Firebase にアップロード → どこからでもURLで見れる
```

---

## 3ステップの解説

### ステップ1：ビルド（npm run build）

アプリのソースコード（.jsx ファイル）は、そのままではブラウザで動きません。
「ビルド」は、ソースコードをブラウザが理解できる形（HTML/CSS/JS）に変換する作業です。

- 変換前：`src/` フォルダの中のファイル（開発用）
- 変換後：`dist/` フォルダの中のファイル（公開用）

### ステップ2：ログイン（firebase login）

Firebase は Google のサービスです。
「このアプリは自分のものです」と証明するために、Googleアカウントでログインします。
※一度ログインすれば、しばらくは再ログイン不要です。

### ステップ3：デプロイ（firebase deploy）

`dist/` フォルダの中身を Firebase のサーバーにアップロードします。
アップロードが終わると、URLが使えるようになります。

```
https://inspection-time-c4fd3.web.app
```

## 実際のやり方

### 方法1：deploy.bat をダブルクリック（推奨）

フォルダの中にある `deploy.bat` をダブルクリックするだけ。
上の3ステップを全部自動でやってくれます。

### 方法2：手動でやる場合

コマンドプロンプト（スタートメニューで「cmd」と検索）を開いて、
以下を1行ずつ入力してEnter：

```
cd /d "C:\Users\anrw3\.gemini\antigravity\playground\golden-meteoroid"
npm run build
npx firebase deploy --only hosting
```

---

## いつデプロイが必要？

コードを修正したら、デプロイし直さないと公開版には反映されません。

- コードを修正した → deploy.bat をダブルクリック → 公開版に反映
- コードを修正していない → デプロイ不要

---

## 関連ファイルの役割

| ファイル | 何をするファイル？ |
|---------|----------------|
| `deploy.bat` | ダブルクリックでデプロイを自動実行 |
| `firebase.json` | 「distフォルダを公開する」という設定 |
| `.firebaserc` | 「どのFirebaseプロジェクトに公開するか」の設定 |
| `dist/` フォルダ | ビルド後の公開用ファイルが入る場所 |

## トラブル時

| 症状 | 対処 |
|------|------|
| ログイン画面が出ない | ブラウザが裏に隠れている可能性あり。タスクバーを確認 |
| ビルドが失敗 | コードにエラーがある。npm run dev で先に動作確認する |
| デプロイが失敗 | ネットに繋がっているか確認。再度 deploy.bat を実行 |
